## 1. 模块概述

### 1.1 定位

在现有业务系统之上叠加 AI 决策辅助层，接入小米 MiMo API，将系统内的结构化警务数据（文字）与非结构化数据（警员上报的图片/视频/音频）转化为 AI 可理解的上下文，辅助指挥员做出更快、更优的决策。

**核心原则：AI 建议仅供参考，所有操作需人工确认后执行。**

### 1.2 扩展能力对比（MiniMax → MiMo）

| 能力 | 原方案（MiniMax） | 新方案（MiMo） |
|------|-----------------|--------------|
| 文本推理 | ✅ | ✅ 更强（万亿参数，1M context） |
| 图片理解 | ❌ | ✅ MiMo-V2.5-Omni |
| 视频理解 | ❌ | ✅ 最长 1 小时视频 |
| 语音识别（ASR） | ❌ | ✅ MiMo-V2.5-ASR |
| 语音合成（TTS） | ❌ | ✅ MiMo-V2.5-TTS（当前免费） |
| 音色克隆 | ❌ | ✅ MiMo-V2.5-TTS-VoiceClone |
| 流式文字输出 | ✅ SSE | ✅ SSE |
| OpenAI SDK 兼容 | ✅ | ✅ 直接复用 |

## 2. MiMo 模型体系

### 2.1 API 基础信息

| 项目 | 值 |
|------|-----|
| Base URL | `https://api.xiaomimimo.com/v1` |
| 认证方式 | Header: `api-key: $MIMO_API_KEY` |
| SDK 兼容 | OpenAI Python SDK / JS SDK（`base_url` 替换即可） |
| 计费方式 | Token Plan — 按 input/output token 总数计费 |
| TTS 计费 | **当前限时免费** |

```python
from openai import OpenAI

client = OpenAI(
    api_key=os.environ["MIMO_API_KEY"],
    base_url="https://api.xiaomimimo.com/v1"
)
```

### 2.2 模型清单

| Model ID | 类型 | 上下文 | 特点 |
|----------|------|--------|------|
| `mimo-v2.5-pro` | 文本推理 | 1M token | 万亿参数，Agent 基座，逻辑推理最强 |
| `mimo-v2.5-omni` | 全模态 | 256K token | 图片/视频（≤1h）/音频（≤10h）/文本统一输入 |
| `mimo-v2.5-asr` | 语音识别 | — | 音频/视频 → 文字，支持多语言 |
| `mimo-v2.5-tts` | 语音合成 | — | 预置音色，支持方言/情绪/唱歌 |
| `mimo-v2.5-tts-voicedesign` | 语音合成 | — | 文本描述定制音色 |
| `mimo-v2.5-tts-voiceclone` | 语音合成 | — | 音频样本复刻音色 |

## 3. 功能规划与模型对应

### 3.1 阶段一：智能文字辅助（使用 mimo-v2.5-pro）

**适用场景：纯文字输入输出，改造成本最低。**

| 功能 | 模型 | 入口 |
|------|------|------|
| 智能排班推荐 | `mimo-v2.5-pro` | 巡逻调度 → 派发任务 → 「AI 推荐警员」 |
| 装备推荐 | `mimo-v2.5-pro` | 巡逻调度 / 装备管理 → 「AI 推荐装备」 |
| 警情研判 | `mimo-v2.5-pro` | 统计分析 → AI 研判 Tab |
| 自然语言数据查询 | `mimo-v2.5-pro` | 全局悬浮 AI 助手 |

### 3.2 阶段二：多模态能力（使用 mimo-v2.5-omni / asr）

**适用场景：警员上报现场图片/视频，或语音录入报警信息。**

| 功能 | 模型 | 入口 | 说明 |
|------|------|------|------|
| 现场图片自动描述 | `mimo-v2.5-omni` | 案件证据 → 上传图片后自动生成描述 | 减少手动填写 |
| 巡逻视频摘要 | `mimo-v2.5-omni` | 巡逻任务 → 上传巡逻视频 → 生成文字小结 | 替代手写汇报 |
| 语音接警录入 | `mimo-v2.5-asr` | 报警受理 → 语音输入模式 | 口述警情 → 自动填表 |
| 语音上报巡逻情况 | `mimo-v2.5-asr` | 巡逻调度 → 打卡签到 → 语音汇报 | 现场口头上报 → 文字存档 |

### 3.3 阶段三：语音播报（使用 mimo-v2.5-tts）

**适用场景：系统主动播报告警、调度通知。**

| 功能 | 模型 | 说明 |
|------|------|------|
| 紧急警情语音播报 | `mimo-v2.5-tts` | 新接到 3级/4级紧急警情时自动语音提醒 |
| 调度通知语音播报 | `mimo-v2.5-tts` | 给被派任务的警员播报任务信息 |
| 统计报表播报 | `mimo-v2.5-tts` | 每日0点/8点播报当日统计简报 |

## 4. Prompt 工程设计

### 4.1 智能排班推荐

```
你是一个智能警务调度助手，请基于以下实时数据为即将派发的巡逻任务推荐最合适的警员。

【任务信息】
类型：{taskType}（例行巡逻/专项检查/定点值守）
区域：{areaName}
时间：{taskStart} ~ {taskEnd}
紧急程度：{urgency}（1-4级）

【当前在岗警员】（共 {count} 人）
{officerTable}
（格式：警号 | 姓名 | 类别 | 擅长 | 近30天任务数 | 距上次执勤天数）

【该时段已排班警员（排除）】
{scheduledOfficers}

【该区域近30天主要案件类型 Top5】
{areaCaseTypes}

请以 JSON 格式回答（仅输出 JSON，不含其他文字）：
{"recommended":[{"officerId":数字,"name":"姓名","reason":"理由"}],"count":建议人数,"summary":"整体说明"}
```

**Token 控制**：上下文压缩到 800 token 以内，警员信息每人 1 行，案件类型只传 Top5。

### 4.2 装备推荐

```
你是警务装备推荐助手，请根据任务和区域历史情况推荐携带装备。

【任务】类型：{taskType}，时段：{shiftType}，区域：{areaName}
【该区域近期多发案件】{topCaseTypes}
【当前空闲装备】{availableEquipment}
（格式：名称 | 类型 | 可用数）

以 JSON 回答：
{"must":[{"name":"名称","reason":"原因"}],"suggested":[{"name":"名称","reason":"原因"}],"summary":"说明"}
```

### 5.4 SSE 流式 Controller 示例

```java
@PostMapping(value = "/ai/schedule/recommend",
             produces = MediaType.TEXT_EVENT_STREAM_VALUE)
public SseEmitter scheduleRecommend(@RequestBody AiScheduleRequest req) {
    SseEmitter emitter = new SseEmitter(60_000L);
    CompletableFuture.runAsync(() -> {
        try {
            String prompt = aiScheduleService.buildPrompt(req);
            mimoClient.streamChat(
                "mimo-v2.5-pro", prompt, "请给出推荐",
                token -> emitter.send(SseEmitter.event().data(token)),
                () -> emitter.complete()
            );
        } catch (Exception e) {
            emitter.completeWithError(e);
        }
    });
    return emitter;
}
```

### 6.5 前端 API 文件（src/api/ai.js）

```javascript
import axios from 'axios'

const authHeader = () => ({
  Authorization: `Bearer ${localStorage.getItem('accessToken')}`
})

export const aiApi = {
  // 文字类（SSE）
  scheduleRecommend: (data, onToken, onDone) => ssePost('/ai/schedule/recommend', data, onToken, onDone),
  equipmentRecommend: (data) => axios.post('/api/ai/equipment/recommend', data, { headers: authHeader() }),
  analysisChat: (messages, onToken, onDone) => ssePost('/ai/analysis/chat', { messages }, onToken, onDone),
  chat: (messages, onToken, onDone) => ssePost('/ai/chat', { messages }, onToken, onDone),

  // 多模态（Omni）
  describeImage: (base64Image) => axios.post('/api/ai/omni/describe-image', { image: base64Image }, { headers: authHeader() }),
  summarizeVideo: (videoUrl) => axios.post('/api/ai/omni/summarize-video', { videoUrl }, { headers: authHeader() }),

  // 语音（ASR）
  transcribeAudio: (base64Audio) => axios.post('/api/ai/asr/transcribe', { audio: base64Audio }, { headers: authHeader() }),

  // 语音（TTS）
  speak: (text, voice = '苏打') => axios.post('/api/ai/tts/speak', { text, voice }, {
    headers: authHeader(), responseType: 'arraybuffer'
  }),

  // 历史
  history: (params) => axios.get('/api/ai/history', { params, headers: authHeader() })
}

function ssePost(path, body, onToken, onDone) {
  // 用 fetch SSE，携带 token
  fetch(`/api${path}`, {
    method: 'POST',
    headers: { 'Content-Type': 'application/json', ...authHeader() },
    body: JSON.stringify(body)
  }).then(async res => {
    const reader = res.body.getReader()
    const decoder = new TextDecoder()
    while (true) {
      const { done, value } = await reader.read()
      if (done) { onDone?.(); break }
      const text = decoder.decode(value)
      text.split('\n').filter(l => l.startsWith('data:')).forEach(l => {
        const d = l.slice(5).trim()
        if (d && d !== '[DONE]') onToken?.(d)
      })
    }
  })
}
```

## 7. 数据库设计

### 7.1 AI 对话历史表

```sql
CREATE TABLE ai_conversation (
    id            BIGINT UNSIGNED NOT NULL AUTO_INCREMENT,
    scene         VARCHAR(20)     NOT NULL COMMENT 'schedule/equipment/analysis/chat/asr/tts/omni',
    model_used    VARCHAR(50)     NOT NULL COMMENT '实际调用的模型ID',
    user_id       BIGINT UNSIGNED NOT NULL,
    input_tokens  INT             NOT NULL DEFAULT 0,
    output_tokens INT             NOT NULL DEFAULT 0,
    prompt_summary VARCHAR(500)            COMMENT '上下文摘要（不存完整prompt节省空间）',
    response_text TEXT                     COMMENT 'AI响应',
    latency_ms    INT,
    is_adopted    TINYINT(1)      DEFAULT 0 COMMENT 'AI建议是否被采纳',
    created_at    DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    created_by    BIGINT UNSIGNED,
    PRIMARY KEY (id),
    KEY idx_scene_user (scene, user_id),
    KEY idx_created_at (created_at)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='AI对话历史';
```

## 10. 风险与降级策略

| 风险 | 应对 |
|------|------|
| API 调用延迟（1-3s） | 流式输出改善体感；非关键功能异步后台执行 |
| Token 成本 | 上下文严格压缩（目标 <800 token/次）；TTS 当前免费，后续按需控量 |
| 模型 JSON 格式不稳定 | try-catch 解析；失败时降级展示原始文本，不报错 |
| 多模态文件过大 | 图片压缩至 ≤1MB；视频限制 ≤10 分钟；超限提示用户 |
| ASR 识别率不足 | 识别结果供人工修改，不强制自动填入 |
| API 不可用 | 熔断降级：AI 入口标灰「服务暂不可用」，主业务流程不受影响 |
| 敏感数据泄露 | Prompt 中不传身份证/手机号等 PII；视频/音频本地转换后仅传 base64 片段 |
