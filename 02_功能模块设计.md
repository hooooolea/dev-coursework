## 1. 系统管理模块

### 1.1 用户管理

**功能列表**

- 新增/编辑/删除警员账号
- 重置密码（BCrypt 加密）
- 账号启用 / 禁用
- 分配角色
- 查询用户列表（支持按姓名、警号、部门筛选）

**关键字段**：警号（唯一）、姓名、手机号、所属单位、账号状态、最后登录时间

### 1.2 角色与权限管理

**角色列表**（系统内置，可扩展）

| 角色 | 说明 |
|------|------|
| ROLE_SUPER_ADMIN | 超级管理员，不受权限限制 |
| ROLE_DIRECTOR | 局长/所长，全局只读+报表 |
| ROLE_OFFICER | 民警，日常业务操作 |
| ROLE_AUXILIARY | 辅警，受限操作 |
| ROLE_DUTY | 值班员，报警受理 |

**权限粒度**：菜单权限（能否看到该菜单）+ 操作权限（能否执行增删改查）+ 数据权限（能看到哪些数据）

**权限编码示例**：

```
case:view      案件查询
case:add       案件新增
case:edit      案件编辑
case:delete    案件删除
case:export    案件导出
person:view    人员查询
vehicle:view   车辆查询
...
```

### 1.3 数据字典管理

维护系统内全部枚举值，避免硬编码：
- 案件类型（刑事案件、治安案件、交通事故等）
- 案件状态（待立案、侦查中、已移送、已结案等）
- 人员类型（普通市民、重点人员、在逃人员等）
- 证件类型（身份证、护照、军官证等）
- 支持在线新增/修改字典项，实时生效

### 1.4 部门管理

- 树形结构展示（市局 → 分局 → 派出所）
- 新增/编辑/删除部门节点
- 关联用户归属

### 1.5 操作日志

- 记录所有写操作（新增、修改、删除）
- 查询条件：操作人、模块、时间范围、操作类型
- 日志只读，不可删除
- 支持导出 Excel

## 2. 报警受理模块

### 2.1 接警录入

接到 110 报警后，值班员录入警情信息：

**必填信息**
- 报警时间（自动填入当前时间）
- 报警人姓名、联系电话
- 事发地址（省市区 + 详细地址）
- 警情类型（从字典选择）
- 警情描述（自由文本）
- 紧急程度（一般 / 较紧急 / 紧急 / 特急）

**可选信息**
- 涉案人员（关联人员档案）
- 涉案车辆（关联车辆档案）
- 录音附件

### 2.2 任务派发

接警后自动或手动派发处置任务：

- 查看当前在岗、在线警员列表
- 选择一名或多名警员接警处置
- 发送站内消息通知
- 设置预计到达时间
- 任务状态：待接收 → 已接收 → 处置中 → 已处置

### 2.3 警情跟踪

- 处置警员实时更新处置进展
- 到达现场打卡（记录时间）
- 填写处置结果
- 关联生成案件（若需要立案）
- 警情关闭，填写结果摘要

### 2.4 警情列表

- 按状态、时间、类型、紧急程度筛选
- 超时未处置警情高亮显示
- 当日警情汇总统计（接警数、处置数、平均响应时长）

**核心接口**

```
POST   /api/alarm/create          接警录入
PUT    /api/alarm/{id}/dispatch   任务派发
PUT    /api/alarm/{id}/arrive     到达现场
PUT    /api/alarm/{id}/close      警情关闭
GET    /api/alarm/list            警情列表
GET    /api/alarm/{id}            警情详情
```

## 3. 案件管理模块

### 3.1 案件立案

**立案信息**
- 案件编号（系统自动生成，规则：AJYYYYMMDDNNN）
- 案件名称
- 案件类型（刑事 / 治安 / 行政）
- 案件子类型（盗窃、抢劫、诈骗、斗殴等，从字典选择）
- 发案时间、发案地点
- 案情描述
- 立案民警（默认当前用户）
- 立案时间（自动）
- 关联报警记录（可选）

### 3.2 案件侦查

**侦查记录**（每次可新增多条进展）
- 侦查时间
- 侦查内容（自由文本）
- 负责民警
- 下一步计划

**证据管理**
- 上传证据文件（图片、视频、文档）
- 证据编号、证据类型、描述
- 证据收集时间、收集人
- 证据存放位置
- 证据链完整性校验

**嫌疑人管理**
- 关联人员档案
- 标注嫌疑程度（嫌疑人 / 重要嫌疑人 / 已确认）
- 添加审讯记录

### 3.3 案件状态流转

```
待立案 ──▶ 侦查中 ──▶ 已移送检察院 ──▶ 已结案
                │
                └──▶ 撤案（附撤案原因）
```

- 每次状态变更记录变更时间、操作人、变更原因
- 关键节点（移送、结案）需填写审批备注

### 3.4 案件查询

- 多条件组合筛选：时间范围、类型、状态、负责人、地区
- 关键词全文搜索（案件名称、案情描述）
- 列表排序：按立案时间、案件等级
- 超期未结案件标红提醒（超过60天）

### 3.5 案件关联分析

- 同一嫌疑人涉及的全部案件
- 同一地点的历史案件
- 同一时间段的同类案件

**核心接口**

```
POST   /api/case/create              立案
PUT    /api/case/{id}/update         更新案件信息
PUT    /api/case/{id}/status         变更案件状态
POST   /api/case/{id}/progress       新增侦查进展
POST   /api/case/{id}/evidence       上传证据
POST   /api/case/{id}/suspect        添加嫌疑人
GET    /api/case/list                案件列表
GET    /api/case/{id}/detail         案件详情
GET    /api/case/export              导出Excel
```

## 4. 人员档案模块

### 4.1 人口信息管理

**基本信息**
- 姓名、性别、民族
- 身份证号（脱敏显示）
- 出生日期
- 户籍地址、现居住地址
- 联系电话（脱敏显示）
- 照片（上传存储）
- 录入时间、录入民警

**扩展信息**
- 职业、工作单位
- 家庭成员关联
- 备注信息

### 4.2 人员分类管理

| 人员类型 | 说明 |
|---------|------|
| 普通人员 | 无特殊标注 |
| 重点关注 | 有违法记录，需重点关注 |
| 在逃人员 | 列入追逃名单 |
| 社区矫正 | 缓刑、假释等人员 |
| 涉毒人员 | 涉毒前科或吸毒记录 |

- 标注人员类型时记录原因、标注民警
- 重点人员系统自动发出预警通知

### 4.3 违法记录

关联人员的历史违法、处罚记录：
- 违法时间、违法类型
- 处理结果（警告、罚款、拘留、起诉）
- 关联案件编号
- 经办民警

### 4.4 人员查询

- 精确查询：身份证号、姓名
- 模糊查询：姓名关键词、地址关键词
- 分类筛选：人员类型、地区
- 照片比对（预留接口，可集成人脸识别）

**核心接口**

```
POST   /api/person/create          新增人员
PUT    /api/person/{id}            更新人员信息
POST   /api/person/{id}/label      标注人员类型
POST   /api/person/{id}/violation  新增违法记录
GET    /api/person/list            人员列表
GET    /api/person/{id}            人员详情
GET    /api/person/{id}/cases      涉及案件
```

## 5. 车辆管理模块

### 5.1 车辆档案

**基本信息**
- 车牌号（唯一）
- 车架号（VIN）
- 车辆类型（轿车、SUV、货车、摩托车等）
- 品牌型号、颜色、年款
- 登记人（关联人员档案）
- 登记日期、有效期至
- 车辆照片

**状态标注**
- 正常 / 可疑 / 布控 / 已扣押

### 5.2 违章记录

- 违章时间、地点
- 违章类型（超速、闯红灯、违停等）
- 罚款金额、扣分
- 是否已处理
- 关联交通事故案件

### 5.3 布控管理

- 对可疑车辆设置布控
- 布控原因、布控级别（一般 / 紧急）
- 布控时间范围
- 布控期间过车预警（预留卡口接口）
- 解控记录

### 5.4 车辆追踪

- 关联案件（如肇事逃逸）
- 追踪状态、追踪记录
- 找回登记

**核心接口**

```
POST   /api/vehicle/create           登记车辆
PUT    /api/vehicle/{id}             更新车辆信息
POST   /api/vehicle/{id}/violation   新增违章
POST   /api/vehicle/{id}/control     设置布控
PUT    /api/vehicle/{id}/decontrol   解除布控
GET    /api/vehicle/list             车辆列表
GET    /api/vehicle/{id}             车辆详情
GET    /api/vehicle/search?plate=    按车牌查询
```

## 6. 巡逻调度模块

### 6.1 排班管理

- 按周/月排班
- 支持早班、中班、夜班
- 班次配置（时间段、人员最少要求）
- 调班申请与审批
- 导出排班表（Excel）

### 6.2 区域划分

- 配置辖区内巡逻网格/区域
- 每个区域设定责任警员/小组
- 区域重要等级标注

### 6.3 任务管理

- 派发巡逻任务（指定人员、区域、时间）
- 任务类型：例行巡逻、专项检查、定点值守
- 任务接收确认
- 巡逻中汇报（填写情况说明）
- 任务完成提交（填写巡逻小结）

### 6.4 打卡签到

- 巡逻开始/结束打卡
- 记录打卡时间
- 预留 GPS 坐标字段（可对接移动端）
- 未按时打卡自动提醒

### 6.5 出勤统计

- 警员出勤率统计
- 巡逻里程/时长统计
- 迟到/早退记录
- 月度出勤报表

**核心接口**

```
POST   /api/patrol/schedule/create    新建排班
GET    /api/patrol/schedule/week      本周排班
POST   /api/patrol/task/create        派发任务
PUT    /api/patrol/task/{id}/accept   接收任务
PUT    /api/patrol/task/{id}/checkin  签到打卡
PUT    /api/patrol/task/{id}/complete 完成任务
GET    /api/patrol/task/my            我的任务
```

## 7. 警力资源模块

### 7.1 警员档案

**个人信息**
- 警号（工号）、姓名、性别
- 警察类别（刑警、交警、治安警等）
- 警衔/职务
- 所属单位、入职时间
- 联系方式、紧急联系人

**业务信息**
- 擅长领域（可多选）
- 资质证书
- 当前状态（在岗、休假、外出公干、停职）

### 7.2 绩效考核

- 按月 / 季度 / 年度考核
- 考核维度：出勤率、案件参与、违规记录、表彰奖励
- 考核结果：优秀/良好/合格/不合格
- 考核历史查询

### 7.3 装备物资管理

**装备档案**
- 装备编号、名称、型号
- 装备类型（武器、车辆、通信设备、防护装备等）
- 购置日期、使用年限、当前状态

**出入库管理**
- 装备领用申请（申请人、装备、用途）
- 审批流程
- 归还登记
- 库存实时统计
- 装备报废申请

**核心接口**

```
GET    /api/officer/list                 警员列表
GET    /api/officer/{id}                 警员详情
PUT    /api/officer/{id}/status          更新状态
POST   /api/officer/{id}/assessment      新增考核
POST   /api/equipment/borrow            装备领用
POST   /api/equipment/return            装备归还
GET    /api/equipment/inventory         库存查询
```

## 9. 接口权限总览

### 9.1 权限矩阵

| 模块 | 超级管理员 | 局长 | 民警 | 辅警 | 值班员 |
|------|:---:|:---:|:---:|:---:|:---:|
| 系统管理 | ✅ 全部 | ❌ | ❌ | ❌ | ❌ |
| 报警受理 | ✅ | 👁 | 👁 | 👁 | ✅ 全部 |
| 案件管理 | ✅ | 👁 | ✅ 本所 | 📝 录入 | 👁 |
| 人员档案 | ✅ | 👁 | ✅ | 📝 辅助 | 👁 |
| 车辆管理 | ✅ | 👁 | ✅ | 📝 辅助 | 👁 |
| 巡逻调度 | ✅ | 👁 | ✅ | ✅ 执行 | ❌ |
| 警力资源 | ✅ | 👁 | 👁 个人 | 👁 个人 | ❌ |
| 统计分析 | ✅ | ✅ | 📊 本所 | ❌ | 📊 值班 |

> ✅全权限  👁只读  📝有限写入  📊有限统计  ❌无权限

### 9.2 数据权限规则

- **民警**：默认只能查看和操作本派出所数据
- **局长**：可查看全局数据，但不能修改
- **超级管理员**：无限制
- 跨所调阅需填写申请原因，并记录日志

### 9.3 接口规范

- 所有接口路径前缀：`/api/`
- 鉴权接口返回 401，权限不足返回 403
- 列表接口统一支持分页参数：`page`（页码，从1开始）、`size`（每页条数，默认20，最大100）
- 时间字段统一使用 ISO 8601 格式：`yyyy-MM-dd HH:mm:ss`
- 文件上传接口：`Content-Type: multipart/form-data`

## 10. AI 智能辅助模块

> 详细设计见 **05_AI智能板块设计.md**，本节为功能概览。

### 10.1 模块定位

接入 MiniMax 大模型 API（Token Plan），将系统内结构化数据转化为 AI 可理解的上下文，提供四类智能辅助功能。所有 AI 建议仅供参考，需人工确认后执行。

### 10.2 功能列表

| 功能 | 入口位置 | 说明 |
|------|---------|------|
| 智能排班推荐 | 巡逻调度 → 派发任务 | 根据任务类型/区域/时间推荐最优警员组合 |
| 装备推荐 | 巡逻调度 → 派发任务 / 装备管理 | 根据任务和历史案件推荐携带装备 |
| 警情研判助手 | 统计分析 → AI 研判 Tab | 近期警情规律分析、风险区域预测 |
| 自然语言查询 | 全局悬浮 AI 助手 | 用自然语言查询系统数据 |

### 10.3 核心接口

```
POST  /api/ai/schedule/recommend   智能排班推荐（SSE流式）
POST  /api/ai/equipment/recommend  装备推荐（JSON）
POST  /api/ai/analysis/chat        警情研判对话（SSE流式）
POST  /api/ai/chat                 通用查询（SSE流式）
GET   /api/ai/history              AI对话历史
POST  /api/ai/adoption             提交采纳反馈
```

### 10.4 权限控制

| 角色 | 排班推荐 | 装备推荐 | 研判助手 | 自然语言查询 |
|------|:-------:|:-------:|:-------:|:----------:|
| 超级管理员 | ✅ | ✅ | ✅ | ✅ |
| 局长 | ✅ | ✅ | ✅ | ✅ |
| 民警 | ✅ | ✅ | 👁只读 | ✅ |
| 辅警 | ❌ | ✅ | ❌ | ✅ |
| 值班员 | ❌ | ❌ | ❌ | ✅ |
