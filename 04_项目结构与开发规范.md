## 2. 包结构说明

### 2.1 分层约定

| 包名 | 职责 | 命名规范 |
|------|------|---------|
| `controller` | 接收请求，返回响应 | `XxxController` |
| `service` | 业务逻辑接口 | `XxxService` |
| `service.impl` | 业务逻辑实现 | `XxxServiceImpl` |
| `mapper` | 数据库操作接口 | `XxxMapper` |
| `entity` | 数据库实体类 | `XxxEntity` 或直接 `Xxx` |
| `dto` | 请求/响应数据传输对象 | `XxxDTO`（请求）、`XxxVO`（响应） |

### 2.2 实体类示例

```java
@Data
@TableName("case_info")
public class CaseInfo {

    @TableId(type = IdType.AUTO)
    private Long id;

    private String caseNo;
    private String caseName;
    private String caseCategory;
    private String caseType;
    private LocalDateTime occurredAt;
    private String locationDetail;
    private String caseDesc;
    private String status;
    private Integer severityLevel;
    private Long leadOfficerId;
    private Long deptId;

    @TableField(fill = FieldFill.INSERT)
    private LocalDateTime createdAt;

    @TableField(fill = FieldFill.INSERT_UPDATE)
    private LocalDateTime updatedAt;

    @TableField(fill = FieldFill.INSERT)
    private Long createdBy;

    @TableLogic
    private Integer isDeleted;
}
```

### 2.3 统一响应格式实现

```java
@Data
@AllArgsConstructor
@NoArgsConstructor
public class Result<T> {

    private Integer code;
    private String message;
    private T data;
    private Long timestamp = System.currentTimeMillis();

    public static <T> Result<T> ok(T data) {
        return new Result<>(200, "success", data, System.currentTimeMillis());
    }

    public static <T> Result<T> ok() {
        return ok(null);
    }

    public static <T> Result<T> fail(String message) {
        return new Result<>(500, message, null, System.currentTimeMillis());
    }

    public static <T> Result<T> fail(Integer code, String message) {
        return new Result<>(code, message, null, System.currentTimeMillis());
    }
}
```

## 5. 编码规范

### 5.1 命名规范

- **类名**：大驼峰，`CaseInfoController`
- **方法名/变量名**：小驼峰，`getCaseDetail`、`caseNo`
- **常量**：全大写下划线，`MAX_LOGIN_FAIL_COUNT`
- **数据库表名**：小写蛇形，`case_info`
- **接口路径**：小写中划线，`/api/case-info/list`（或下划线均可，保持统一）

### 5.2 Controller 规范

```java
@RestController
@RequestMapping("/api/case")
@RequiredArgsConstructor
public class CaseController {

    private final CaseService caseService;

    @GetMapping("/list")
    @PreAuthorize("hasAuthority('case:view')")
    public Result<IPage<CaseVO>> list(@Valid CaseQueryDTO query) {
        return Result.ok(caseService.listPage(query));
    }

    @PostMapping("/create")
    @PreAuthorize("hasAuthority('case:add')")
    @OperationLog(module = "案件管理", action = "新增案件")
    public Result<Long> create(@RequestBody @Valid CaseCreateDTO dto) {
        return Result.ok(caseService.createCase(dto));
    }
}
```

### 5.3 Service 规范

- 接口与实现分离，接口定义在 `service` 包，实现在 `service.impl` 包
- 事务从 Service 层开始，Controller 层不加 `@Transactional`
- 业务校验抛出 `BusinessException`，不直接返回 null

```java
@Service
@RequiredArgsConstructor
@Transactional(rollbackFor = Exception.class)
public class CaseServiceImpl implements CaseService {

    private final CaseInfoMapper caseInfoMapper;
    private final RedisTemplate<String, Object> redisTemplate;

    @Override
    public Long createCase(CaseCreateDTO dto) {
        // 1. 业务校验
        // 2. 生成案件编号
        String caseNo = generateCaseNo();
        // 3. 保存数据
        CaseInfo caseInfo = BeanUtil.copyProperties(dto, CaseInfo.class);
        caseInfo.setCaseNo(caseNo);
        caseInfoMapper.insert(caseInfo);
        // 4. 清除相关缓存
        return caseInfo.getId();
    }
}
```

### 5.5 异常处理规范

```java
// 自定义业务异常
public class BusinessException extends RuntimeException {
    private final Integer code;

    public BusinessException(String message) {
        super(message);
        this.code = 500;
    }

    public BusinessException(Integer code, String message) {
        super(message);
        this.code = code;
    }
}

// 全局异常处理
@RestControllerAdvice
@Slf4j
public class GlobalExceptionHandler {

    @ExceptionHandler(BusinessException.class)
    public Result<?> handleBusiness(BusinessException e) {
        log.warn("业务异常: {}", e.getMessage());
        return Result.fail(e.getCode(), e.getMessage());
    }

    @ExceptionHandler(MethodArgumentNotValidException.class)
    public Result<?> handleValidation(MethodArgumentNotValidException e) {
        String msg = e.getBindingResult().getFieldErrors().stream()
            .map(FieldError::getDefaultMessage)
            .collect(Collectors.joining("; "));
        return Result.fail(400, msg);
    }

    @ExceptionHandler(AccessDeniedException.class)
    public Result<?> handleAccessDenied(AccessDeniedException e) {
        return Result.fail(403, "无权限访问");
    }

    @ExceptionHandler(Exception.class)
    public Result<?> handleException(Exception e) {
        log.error("系统异常", e);
        return Result.fail(500, "系统内部错误，请联系管理员");
    }
}

## 6. 开发流程建议

### 6.1 新增一个功能模块的标准步骤

```
1. 创建数据库表（写 SQL 文件）
2. 创建 Entity 实体类（对应表字段）
3. 创建 Mapper 接口（继承 BaseMapper<Entity>）
4. 创建 Mapper XML（复杂查询）
5. 创建 DTO（入参）和 VO（出参）
6. 创建 Service 接口 + ServiceImpl
7. 创建 Controller，编写 RESTful 接口
8. 配置权限（sys_permission 表 + @PreAuthorize 注解）
9. 创建 Thymeleaf 页面模板
10. 编写单元测试
```

### 6.2 Git 分支策略

```
main          # 生产分支，只接受 merge request
├── develop   # 开发集成分支
│   ├── feature/case-module      # 功能分支
│   ├── feature/person-module
