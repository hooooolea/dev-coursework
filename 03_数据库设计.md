## 1. 设计规范

### 1.1 命名规范

- 表名、字段名全部使用**小写蛇形命名**（snake_case）
- 表名使用业务前缀（`sys_`、`case_`、`person_`、`vehicle_`、`patrol_`）
- 主键统一命名为 `id`，类型 `BIGINT UNSIGNED AUTO_INCREMENT`
- 外键字段命名为 `{关联表}_id`
- 时间字段：`created_at`（创建时间）、`updated_at`（更新时间）、`deleted_at`（软删时间）

### 1.2 通用字段约定

所有业务表包含以下基础字段：

```sql
id          BIGINT UNSIGNED NOT NULL AUTO_INCREMENT PRIMARY KEY COMMENT '主键',
created_at  DATETIME        NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
updated_at  DATETIME        NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
created_by  BIGINT UNSIGNED          COMMENT '创建人ID',
is_deleted  TINYINT(1)      NOT NULL DEFAULT 0 COMMENT '是否软删除 0否1是'
```

### 1.3 数据库全局设置

```sql
-- 创建数据库
CREATE DATABASE IF NOT EXISTS smart_police
    CHARACTER SET utf8mb4
    COLLATE utf8mb4_unicode_ci;

USE smart_police;

-- 关闭外键检查（导入时）
SET FOREIGN_KEY_CHECKS = 0;
```

## 3. 系统管理表组

### 3.1 用户表 `sys_user`

```sql
CREATE TABLE sys_user (
    id            BIGINT UNSIGNED  NOT NULL AUTO_INCREMENT COMMENT '主键',
    badge_no      VARCHAR(20)      NOT NULL COMMENT '警号（唯一）',
    username      VARCHAR(50)      NOT NULL COMMENT '登录用户名',
    password      VARCHAR(255)     NOT NULL COMMENT 'BCrypt加密密码',
    real_name     VARCHAR(50)      NOT NULL COMMENT '真实姓名',
    gender        TINYINT(1)       NOT NULL DEFAULT 1 COMMENT '性别 1男2女',
    phone         VARCHAR(20)               COMMENT '手机号（AES加密存储）',
    email         VARCHAR(100)              COMMENT '邮箱',
    dept_id       BIGINT UNSIGNED           COMMENT '所属部门ID',
    avatar        VARCHAR(500)              COMMENT '头像URL',
    status        TINYINT(1)       NOT NULL DEFAULT 1 COMMENT '账号状态 1正常 0禁用',
    login_fail_count INT            NOT NULL DEFAULT 0 COMMENT '连续登录失败次数',
    lock_until    DATETIME                  COMMENT '账号锁定到期时间',
    last_login_at DATETIME                  COMMENT '最后登录时间',
    last_login_ip VARCHAR(45)               COMMENT '最后登录IP',
    remark        VARCHAR(500)              COMMENT '备注',
    created_at    DATETIME         NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    updated_at    DATETIME         NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
    created_by    BIGINT UNSIGNED           COMMENT '创建人ID',
    is_deleted    TINYINT(1)       NOT NULL DEFAULT 0 COMMENT '是否删除',
    PRIMARY KEY (id),
    UNIQUE KEY uk_badge_no (badge_no),
    UNIQUE KEY uk_username (username),
    KEY idx_dept_id (dept_id),
    KEY idx_status (status)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='系统用户表';
```

-- 车牌查询（精确，已有唯一索引 uk_plate_no）

-- 警情按时间+状态查询
ALTER TABLE alarm_record ADD INDEX idx_time_status (alarm_time, status);

-- 操作日志按用户+时间查询
ALTER TABLE sys_operation_log ADD INDEX idx_user_time (user_id, created_at);

### 11.2 分区建议（大数据量）

当操作日志、违章记录等数据超过千万条时，可按月分区：

```sql
-- 操作日志按月分区示例（需建表时指定）
ALTER TABLE sys_operation_log
    PARTITION BY RANGE (YEAR(created_at) * 100 + MONTH(created_at)) (
        PARTITION p202401 VALUES LESS THAN (202402),
        PARTITION p202402 VALUES LESS THAN (202403),
        -- ...
        PARTITION p_future VALUES LESS THAN MAXVALUE
    );
```
