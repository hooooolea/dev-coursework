# 智能警务管理系统 — 大作业报告

## 一、项目概述

### 1.1 项目背景

面向公安机关的数字化警务管理平台，旨在实现警务工作流程化、可追溯化，覆盖案件管理、人员档案、车辆管理、巡逻调度、报警受理、统计分析等核心业务场景，并接入大模型实现 AI 辅助决策。

### 1.2 项目目标

- 建立统一的警务数据中心，消除信息孤岛
- 实现案件全生命周期电子化管理
- 提供实时警情监控与调度能力
- 接入大模型实现智能推荐与警情研判
- 保障数据安全与操作可追溯

### 1.3 用户角色

| 角色 | 权限范围 | 人数 |
|------|---------|:--:|
| 超级管理员 | 系统全部功能 | 1 |
| 局长/所长 | 全局数据查阅、报表导出 | 1 |
| 民警 | 案件、人员、车辆、巡逻日常操作 | 5 |
| 辅警 | 报警录入、巡逻打卡 | 1 |
| 值班员 | 报警受理全权、任务派发 | 1 |

## 三、系统架构

### 3.1 分层架构

```
前端展示层 (Vue 3 + Element Plus + ECharts)
        ↕ HTTP / SSE
Controller 层 (@RestController, 参数校验, 统一响应)
        ↕
Service 层 (业务逻辑, 事务管理, 权限校验)
        ↕
AI 集成层 (大模型 API 调用, Prompt 构建, 上下文采集)
        ↕
DAO / Mapper 层 (MyBatis-Plus, 动态 SQL, 分页)
        ↕
数据存储层 (MySQL 8.0 + Redis 7)
```

### 3.2 核心设计

- **认证流程**：用户名密码 → Spring Security 认证 → 生成 JWT → 前端存储 → 后续请求携带 Token → 过滤器校验
- **权限控制（RBAC）**：User → UserRole → Role → RolePermission → Permission，三级粒度（菜单/接口/数据权限）
- **统一响应**：`Result<T> { code, message, data, timestamp }`，全局异常拦截
- **AOP 日志**：`@OperationLog` 切面记录所有写操作（操作人、IP、耗时）
- **缓存策略**：Redis 管理 Token 黑名单、登录失败计数、接口限流

## 四、数据库设计

### 4.1 整体规模

- 30 张业务表（21 张有演示数据，共 319 条记录），覆盖 9 大功能模块
- 设计规范：小写蛇形命名、统一基础字段（id/created_at/updated_at/is_deleted）、逻辑删除、主键自增

### 4.2 ER 关系（核心表组）

```
sys_user ←→ sys_user_role ←→ sys_role ←→ sys_role_permission ←→ sys_permission
    │
    ├── alarm_record ←→ alarm_dispatch
    ├── case_info ←→ case_progress / case_evidence / case_suspect ←→ person_info
    ├── person_info ←→ person_violation
    ├── vehicle_info ←→ vehicle_violation / vehicle_control
    ├── patrol_schedule ←→ patrol_task ←→ patrol_checkin
    ├── officer_info ←→ officer_assessment
    └── equipment_info ←→ equipment_borrow
```

### 4.3 安全设计

- 密码 BCrypt 加盐哈希存储
- 敏感字段（身份证号、手机号）存储后 4 位，列表脱敏显示
- 软删除（is_deleted）保留数据可追溯性

## 五、功能模块

### 5.1 警情驾驶舱（首页）

- 四维度统计卡片：今日接警、在办案件、布控车辆、在岗警员
- 近 7 天接警趋势折线图（ECharts）
- 案件类型分布饼图
- 最新警情列表
- 加载骨架屏 + 错误重试

### 5.2 报警受理

- 接警录入（报警人、电话、地址、类型、紧急程度、描述）
- 任务派发（选择在岗警员、站内通知）
- 警情跟踪（到达现场打卡 → 处置中 → 关闭）
- 分页筛选 + 超时高亮

### 5.3 案件管理

- 案件立案（自动编号 AJ+日期+序号）
- 侦查进展记录（时间线）
- 证据管理（上传/类型/存放位置，支持多媒体文件）
- 嫌疑人管理（关联人员档案、嫌疑程度标注）
- 状态流转：侦查中 → 已移送 → 已结案 / 撤案

### 5.4 人员档案

- 基本信息 CRUD + 照片上传
- 人员分类（普通/重点关注/在逃/社区矫正/涉毒）
- 违法记录关联
- 身份证+手机号脱敏显示

### 5.5 车辆管理

- 车辆登记（车牌/VIN/品牌型号/颜色/登记人）
- 违章记录管理
- 布控/解控管理
- 状态标注（正常/可疑/布控中/在逃/已扣押）

### 5.6 巡逻调度

- 周视图排班（早/中/夜三班，网格化展示）
- 任务派发 + 签到打卡
- 出勤统计 + 巡逻小结

### 5.7 警力资源

- 警员档案（警号/警种/警衔/职务/擅长领域/在职状态）
- 绩效考核（出勤+案件+奖励−违规=总分，自动评级）
- 装备物资管理（出入库/库存统计）

### 5.8 统计分析

- 案件统计（按月立案趋势柱状图 + 类型饼图 + 状态汇总表）
- 警力效能（在岗率/状态分布/活跃排行 Top10）
- **同比/环比**：立案总数、已结案、破案率的环比（上月 vs 本月）和同比（去年同月）变化百分比
- Excel 报表导出（POI）

## 六、AI 智能辅助模块

### 6.1 整体设计

在业务系统之上叠加 AI 决策辅助层，将结构化警务数据转化为大模型可理解的上下文。核心原则：**AI 建议仅供参考，所有操作需人工确认后执行**。

### 6.2 实现功能

| 功能 | 说明 | 技术 |
|------|------|------|
| 智能排班推荐 | 根据任务/区域/在岗警员/历史案件，AI 推荐最优警员组合 | SSE 流式输出 |
| 装备推荐 | 根据任务类型/区域风险/库存，分级推荐必带和建议装备 | JSON 响应 |
| AI 警情研判 | 汇总近 30 天数据，分析趋势、识别风险区域、给出预防建议 | SSE 流式对话 |
| 全局 AI 助手 | 页面右下角悬浮窗，自然语言查询警务数据 | SSE 流式 |
| 语音接警 | 上传录音文件 / 浏览器实时录音 → ASR 转文字 → 结构化提取 → 自动填表 | ASR + Pro |
| 视频分析 | 上传巡逻视频 → 大模型分析场景/识别可疑/描述特征 | 多模态 Omni |
| TTS 语音播报 | 文字转语音，紧急警情自动播报 | TTS |

### 6.3 技术要点

- API Key 通过 Redis 动态管理，前端配置页输入，不写死在代码中
- SSE 实现流式输出，改善等待体感
- Prompt 模板设计：上下文压缩到 800 token，结构化数据拼接
- 熔断降级：AI 不可用时标灰入口，主业务流程不受影响

## 七、关键技术亮点

### 7.1 Spring Security + JWT 无状态认证

- BCrypt 密码加密，RS256 签名
- 连续登录失败 5 次锁定 30 分钟
- Token 黑名单（Redis）支持主动退出

### 7.2 RBAC 权限控制

- 5 角色 × 60+ 权限细粒度控制
- 接口级 `@PreAuthorize` + 菜单动态加载
- 前端路由守卫 + 请求拦截器 403 自动跳转

### 7.3 SSE 流式响应

- Spring `SseEmitter` + `CompletableFuture` 异步
- OkHttp 逐行读取解析 `data:` 块
- 前端 `fetch` + `ReadableStream` 实时渲染

### 7.4 数据安全

- 敏感字段存储截断（`id_card_tail`/`phone_tail`）
- SQL 预编译防注入
- 文件上传路径防穿越（禁止 `..`）、类型+大小校验

### 7.5 工程化

- 统一响应格式 + 全局异常处理
- AOP 操作日志自动记录
- MyBatis-Plus 逻辑删除 + 自动填充
- Maven 多环境配置（dev/prod）

### 8.5 遇到的问题与解决

| 问题 | 解决方案 |
|------|---------|
| Java 25 不兼容 Lombok 1.18.36 | 降级到 JDK 21 编译运行 |
| MyBatis-Plus `@TableLogic` 要求表有 `is_deleted` 列 | 为 7 张缺失表 ALTER TABLE 补充 |
| 前端表格 `fixed="right"` 导致右侧空白 | 全部改为 `min-width` 自适应撑满 |
| admin 缺 `sys:role:view` 等权限 | 补充 15 条权限记录并分配给超管角色 |
| LoginVO `@Builder` 编译失败 | 改用 `@AllArgsConstructor` + `new LoginVO(...)` |
| BCrypt 哈希与文档密码不匹配 | 重新生成 `123456` 哈希写入数据库 |
| SSE 连接中断导致数据不完整 | 前端 `buffer` 缓存 + 后端 60s 超时 + 异常兜底 |
