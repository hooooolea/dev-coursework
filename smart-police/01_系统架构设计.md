## 1. 项目概述

### 1.1 项目背景

智能警务管理系统面向公安机关，旨在实现警务工作数字化、流程化、可追溯化，覆盖案件管理、人员档案、车辆管理、巡逻调度、报警受理、统计分析等核心业务场景。

### 1.2 系统目标

- 建立统一的警务数据中心，消除信息孤岛
- 实现案件全生命周期电子化管理
- 提供实时警情监控与智能调度能力
- 接入 MiniMax 大模型，实现智能排班推荐、装备推荐与警情研判
- 保障数据安全与操作可追溯

### 1.3 用户角色

| 角色 | 说明 |
|------|------|
| 超级管理员 | 系统配置、用户管理、权限分配 |
| 局长 / 所长 | 全局数据查阅、报表审批、统计分析 |
| 民警 | 案件录入、人员档案管理、日常业务操作 |
| 辅警 | 受限数据录入、巡逻打卡、任务接收 |
| 值班员 | 报警受理、任务派发 |

## 3. 系统分层架构

### 3.1 整体分层

```
┌──────────────────────────────────────────────────────────────┐
│                      前端展示层                               │
│   Vue 3 + Element Plus + ECharts  ←→  SSE 流式接收 AI 响应    │
├──────────────────────────────────────────────────────────────┤
│                    HTTP / HTTPS / SSE 请求                    │
├──────────────────────────────────────────────────────────────┤
│                     过滤器 / 拦截器层                          │
│   Spring Security Filter → JWT验证 → 权限校验 → 限流          │
├──────────────────────────────────────────────────────────────┤
│                    Controller 控制层                           │
│   @RestController → 参数校验 → 响应封装  |  AiController(SSE) │
├──────────────────────────────────────────────────────────────┤
│                     Service 业务层                            │
│   业务逻辑 → 事务管理  |  AiScheduleService / AiEquipService  │
├──────────────────────────────────────────────────────────────┤
│              AI 集成层（com.police.ai）                        │
│   MiniMaxClient → PromptBuilder → 上下文采集 → 对话历史管理    │
│                        ↕ HTTPS / SSE                         │
│              MiniMax API（abab6.5s-chat，Token Plan）          │
├──────────────────────────────────────────────────────────────┤
│                  DAO / Mapper 数据访问层                       │
│   MyBatis-Plus Mapper → 动态SQL → 分页插件                     │
├──────────────────────────────────────────────────────────────┤
│                      数据存储层                               │
│         MySQL 8.0          │         Redis 7               │
└──────────────────────────────────────────────────────────────┘
```

### 3.2 各层职责说明

**Controller 层**
- 接收并解析 HTTP 请求参数
- 调用 `@Valid` / `@Validated` 进行入参校验
- 返回统一响应格式 `Result<T>`
- 不包含任何业务逻辑

**Service 层**
- 承载全部业务逻辑
- 管理事务边界（`@Transactional`）
- 调用多个 Mapper 完成复合业务
- 调用 Redis 进行缓存操作

**Mapper 层**
- 仅负责数据库 CRUD
- 复杂查询通过 XML 编写动态 SQL
- 简单操作使用 MyBatis-Plus 自带方法

### 3.3 统一响应格式

```java
public class Result<T> {
    private Integer code;    // 200成功，400参数错误，401未认证，403无权限，500服务器错误
    private String message;
    private T data;
    private Long timestamp;
}
```

### 3.4 全局异常处理

通过 `@RestControllerAdvice` 统一捕获：
- `BindException` — 参数校验失败
- `BusinessException` — 业务异常（自定义）
- `AccessDeniedException` — 权限不足
- `Exception` — 兜底处理

## 4. 核心架构设计

### 4.1 认证流程

```
用户登录
  │
  ▼
POST /api/auth/login
  │ 校验用户名+密码
  ▼
Spring Security AuthenticationManager
  │ 认证成功
  ▼
生成 JWT Token (accessToken + refreshToken)
  │
  ▼
返回 Token 给前端 → 存储到 localStorage / Cookie
  │
后续请求携带 Authorization: Bearer {token}
  │
  ▼
JwtAuthenticationFilter 拦截 → 解析Token → 载入 SecurityContext
  │
  ▼
Controller 处理请求
```

### 4.2 权限控制（RBAC）

```
User ──── UserRole ──── Role ──── RolePermission ──── Permission
                                                          │
                                                    菜单权限 / 接口权限 / 数据权限
```

- 接口级权限：`@PreAuthorize("hasAuthority('case:edit')")`
- 菜单权限：登录后动态加载菜单树
- 数据权限：民警只能查看本人/本所案件，局长可查全局

### 4.3 缓存策略

| 缓存内容 | Key 规则 | 过期时间 |
|---------|---------|---------|
| 用户Session/Token | `token:{userId}` | 2小时 |
| 菜单权限树 | `menu:{roleId}` | 24小时 |
| 数据字典 | `dict:{type}` | 12小时 |
| 案件统计数据 | `stat:case:{date}` | 1小时 |
| Token黑名单（退出） | `blacklist:{token}` | Token剩余时效 |
| 接口限流计数器 | `rate:{ip}:{api}` | 1分钟 |

### 4.4 AOP 操作日志

使用 `@Aspect` + 自定义注解 `@OperationLog` 切面，记录所有写操作：

```java
@OperationLog(module = "案件管理", action = "新增案件")
public Result<?> addCase(@RequestBody CaseDTO dto) { ... }
```

日志内容：操作人、操作时间、模块、动作、请求参数、IP地址、执行耗时。

### 4.5 定时任务

| 任务 | 触发规则 | 说明 |
|------|---------|------|
| 案件超期预警 | 每天 08:00 | 扫描超期未结案件，推送通知 |
| 日报统计汇总 | 每天 23:55 | 汇总当日警情数据入统计表 |
| Redis缓存预热 | 每天 06:00 | 预加载字典、权限等高频数据 |
| 临时Token清理 | 每小时 | 清理过期黑名单Token |

## 5. 安全架构

### 5.1 身份认证安全

- 密码使用 BCrypt 加盐哈希存储，禁止明文
- JWT 使用 RS256 非对称签名
- 连续登录失败 5 次，账号锁定 30 分钟
- 支持 Token 主动失效（退出后加入黑名单）

### 5.2 数据安全

- 敏感字段（身份证号、手机号）数据库存储加密（AES-256）
- 接口返回时脱敏处理（`150****8888`、`3101**********001X`）
- 所有操作记录审计日志，不可删改

### 5.3 接口安全

- 全局接口限流：同一 IP 每分钟最多 100 次请求
- 关键接口（登录、重置密码）单独限流：每分钟 5 次
- 请求参数防 SQL 注入：MyBatis 预编译 `#{}` 占位符
- 防 XSS：Thymeleaf 默认 HTML 转义

### 5.4 传输安全

- 生产环境强制 HTTPS
- 文件上传校验文件类型、大小限制（≤20MB）
- 下载接口需验证权限，防止越权访问

## 6. 部署架构

### 6.1 开发环境

```
开发者本机
  ├── IntelliJ IDEA
  ├── MySQL 8.0 (本地)
  ├── Redis 7 (本地)
  └── Spring Boot 内嵌 Tomcat (端口 8080)
```

### 6.2 生产环境（单机版）

```
                     ┌──────────────┐
用户浏览器  ──HTTPS──▶  Nginx :443   │
                     │  反向代理     │
                     │  静态资源     │
                     └──────┬───────┘
                            │ HTTP
                            ▼
                     ┌──────────────┐
                     │  Tomcat:8080  │
                     │  Spring Boot  │
                     └──────┬───────┘
                      ┌─────┴─────┐
                      ▼           ▼
               ┌──────────┐  ┌─────────┐
               │ MySQL:3306│  │Redis:6379│
               └──────────┘  └─────────┘
```

### 6.3 Nginx 核心配置

```nginx
server {
    listen 443 ssl;
    server_name police.example.com;

    # 静态资源直接返回
    location /static/ {
        root /opt/police/;
        expires 7d;
    }

    # 动态请求转发
    location / {
        proxy_pass http://127.0.0.1:8080;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }

    # 上传文件大小限制
    client_max_body_size 20m;
}
```

## 7. 非功能性需求

| 指标 | 要求 |
|------|------|
| 响应时间 | 普通查询 < 500ms，报表生成 < 3s |

| 日志保留 | 操作日志保留 1 年，系统日志保留 3 个月 |
| 浏览器兼容 | Chrome 90+、Edge 90+、Firefox 88+ |
