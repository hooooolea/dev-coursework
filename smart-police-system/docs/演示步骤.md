# 智能警务管理系统 — 演示步骤

> 全程 8-10 分钟，覆盖 9 大模块 + AI 功能 + 权限体系

## 准备

确保项目运行：
- 前端 `http://localhost:5174`
- 后端 `http://localhost:8080`
- 已配 MiMo API Key（系统管理 → AI配置）

---

## 阶段一：接警 + AI + 自动立案（2 分钟）

### 步骤 1：登录 + 仪表盘

**操作**：`admin / 123456` 登录

**展示**：
- 5 张统计卡片（今日接警/在办案件/布控车辆/在岗警员/待我处置）——卡片悬浮有上浮动效
- 近 7 天趋势折线图 + 案件类型饼图（全中文标签）
- 最新警情列表

**讲解**："首页仪表盘，实时展示核心警务指标，数据从 MySQL 查询。卡片 hover 有微动效，数字加载有滚动动画。"

### 步骤 2：接警录入 + AI 装备推荐

**操作**：左侧菜单 → **报警受理** → 点击 **演示流程**

**展示**：
- 弹窗自动填入盗窃警情（朝阳区三里屯酒吧街）
- 提交后弹出 AI 分析窗口：加载动画 → 必带/建议装备分级卡片
- 1.5 秒后自动派发给张明（P010）

**讲解**："点击演示流程一键填入报警信息。提交后 AI 异步分析警情，推荐装备方案——必带执法记录仪、对讲机等，建议防刺背心等。同时自动派发给在岗民警。"

### 步骤 3：自动立案

**操作**：左侧菜单 → **案件管理**

**展示**：
- 案件列表出现新记录（派发时自动生成的）
- 案件名称包含类型+地点，状态为"侦查中"
- 回到报警列表，新增"关联"列显示绿色标签"已立案 #ID"

**讲解**："派发后系统自动同步到案件管理，报警和案件通过 relatedCaseId 双向关联。报警列表和案件详情都可看到关联关系。"

---

## 阶段二：案件侦查（2 分钟）

### 步骤 4：案件详情 + 进展

**操作**：案件管理 → 点击任意案件的 **案件详情**

**展示**：
- 统一抽屉打开，顶部绿色条显示"来源警情 BJxxx"
- Tab 1 **侦查进展**：点新增进展 → 填写"勘查现场，提取指纹3枚" → 提交 → 时间线展示
- Tab 2 **证据材料**：点新增证据 → 选择类型+填写描述 → 提交 → 表格展示
- Tab 3 **嫌疑人**：点新增嫌疑人 → 填写姓名/角色(主犯) → 提交 → 列表展示

**讲解**："统一抽屉面板，三个 Tab 管理案件全生命周期。进展以时间线展示，证据支持分类管理，嫌疑人可关联人员档案。"

### 步骤 5：案件状态推进

**操作**：抽屉底部 → 点击 **移送检察院**

**展示**：
- 确认弹窗 → 状态从"侦查中"变"已移送"
- 列表筛选"已移送"可看到该案件

**讲解**："侦查完成后一键移送，状态机完整流转：侦查中→已移送→已结案。"

---

## 阶段三：统计分析 + AI（2 分钟）

### 步骤 6：统计分析

**操作**：左侧菜单 → **统计分析**

**展示**：
- **案件统计 Tab**：4 张卡片（立案总数/已结案/破案率）+ 环比/同比切换 + 趋势柱状图 + 类型饼图
- 切换到 **警力效能 Tab**：在岗率饼图 + 活跃警员排行 + **近30天工作量排行表**

**讲解**："统计分析模块，支持同比环比对比，警员工作量排行基于巡逻任务数计算。数据实时查询数据库。"

### 步骤 7：AI 警情研判

**操作**：左侧菜单 → **AI警情研判**

**展示**：
- 左侧 AI 对话面板，点击快捷问题"近30天警情整体趋势如何"
- AI 流式输出分析报告（**粗体**标题、- 列表、分段）
- 右侧两个饼图（类型分布、区域分布）
- 底部 **语音录入** 按钮 + **上传录音/视频**

**讲解**："AI 研判页，基于近30天真实数据，大模型流式输出分析报告。支持语音录入和视频分析。"

## 阶段四：权限 + 全局助手（2 分钟）

### 步骤 8：全局 AI 助手

**操作**：右下角点击 🤖 按钮

**展示**：
- 展开对话窗口，输入"今天接了多少警情"
- AI 结合当前系统数据回答
- 点击快捷问题"现在有多少警员在岗"

**讲解**："全局悬浮 AI 助手，任意页面可用。AI 会读取系统实时数据回答问题。"

**操作**：退出 → `zhangming / 123456` 登录 → 看菜单 → 退出 → `chengang / 123456` → 看菜单 → 退出 → `sunqiang / 123456` → 看菜单
